# Public-Safety Policy

MetaSkill is intended to be publishable by default. Tracked files should describe reusable workflows, not private local context.

## Do Not Commit

- Private absolute paths.
- Account names, emails, API keys, tokens, calendar details, or service identifiers.
- Unpublished manuscript text, private repository names, private project notes, or reviewer comments.
- Raw logs, traces, prompts, or outputs from private runs.
- Examples that expose local directory layouts or one-off workflows.

## Preferred Fixtures

Use generic placeholders:

- `example-skill`
- `sample-project`
- `private-workspace`
- `references.bib`
- `plan01.md`

## Review Standard

A file is public-safe only when another maintainer can read it without learning private context that is not part of the reusable skill contract.
